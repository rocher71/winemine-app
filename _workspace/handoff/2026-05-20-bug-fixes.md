## A. 발견된 에러 12건 (모두 fix 완료)

### A-1. `react-native-gesture-handler ~3.0.0` 존재하지 않는 버전

- **증상**: `npm install` ERESOLVE — `react-native-gesture-handler@undefined`
- **원인**: spec/`docs/spec/v0.1.0.md` + infra-architect 작성 `package.json`에 `~3.0.0` 명시. 실제 npm stable latest는 **2.31.2** (3.x는 nightly/beta만)
- **fix**: `package.json` → `"react-native-gesture-handler": "~2.28.0"` (expo install이 SDK 54 호환 버전으로 정정)
- **root cause**: spec의 `<libraries>` 섹션에 미존재 버전 기재

### A-2. `lucide-react-native ^0.460.0` react 19 미지원

- **증상**: `npx expo install --fix` ERESOLVE — `peer react@"^16.5.1 || ^17.0.0 || ^18.0.0"`
- **원인**: 0.460.0은 react 19 출시 전 버전. react 19 지원은 **0.523.0+** (0.x 라인 유지하면서 호환)
- **fix**: `package.json` → `"lucide-react-native": "^0.523.0"`
- **검증**: `npm view lucide-react-native@0.523.0 peerDependencies` → react `^19.0.0` 포함

### A-3. `@types/react ^19.1.0` / `typescript ~5.6.0` expo 권장과 불일치

- **증상**: `expo install --fix` 경고 — `expected version: ~19.1.10` / `~5.9.2`
- **fix**: `package.json` → `"@types/react": "~19.1.10"`, `"typescript": "~5.9.2"`

### A-4. `react-native-worklets` 패키지 누락 (Reanimated 4.x 분리)

- **증상**: Metro 번들링 시 `[BABEL]: Cannot find module 'react-native-worklets/plugin'`
- **원인**: Reanimated v4부터 worklets 엔진이 **별도 패키지로 분리**됨. spec/package.json에 deps 누락
- **fix**:
  1. `npx expo install react-native-worklets` 실행 (자동 0.5.1 추가됨)
  2. `babel.config.js` plugin 교체: `'react-native-reanimated/plugin'` → `'react-native-worklets/plugin'`
- **root cause**: spec의 `<libraries>`에 worklets 누락 + babel 설정에 옛 plugin 이름

### A-5. `react-native-url-polyfill` 패키지 누락

- **증상**: Metro 번들링 시 `Unable to resolve "react-native-url-polyfill/auto" from "src/lib/supabase.ts"`
- **원인**: infra-architect가 `src/lib/supabase.ts` 첫 줄에 `import 'react-native-url-polyfill/auto'` 작성 — RN의 URL 객체가 일부 메서드(`searchParams` 등) 누락이라 supabase-js 공식 가이드가 폴리필 권장. 하지만 package.json deps 등록 누락
- **fix**: `npx expo install react-native-url-polyfill` 실행 (자동 추가됨)
- **root cause**: spec의 `<libraries>`에 url-polyfill 누락

### A-6. `babel.config.js` reanimated plugin 옛 이름

- **위치**: `babel.config.js`
- **before**: `plugins: ['react-native-reanimated/plugin']`
- **after**: `plugins: ['react-native-worklets/plugin']`
- **사유**: Reanimated 4.x 공식 권장 (A-4와 연관)

### A-7. `tailwind.config.ts` NativeWind v4 preset 누락

- **증상**: `npx expo start --clear` 시 `Tailwind CSS has not been configured with the NativeWind preset` 에러 (Metro config 로드 실패)
- **원인**: infra-architect가 `tailwind.config.ts` 작성 시 `presets: [nativewindPreset]` 빠뜨림. NativeWind v4 필수
- **fix**:
  ```diff
  + import nativewindPreset from 'nativewind/preset';

    const config: Config = {
      content: [...],
  +   presets: [nativewindPreset],
      darkMode: 'class',
      ...
    };
  ```
- **root cause**: spec의 `<key_implementation_notes>`에 NativeWind 설정 패턴 명시 누락

### A-8. `.env` URL 오타 — `hhttps://` (h 2개)

- **증상**: 런타임 `Error: Invalid supabaseUrl: Must be a valid HTTP or HTTPS URL`
- **원인**: 사용자 직접 입력 실수
- **fix**: 사용자가 `.env` 파일에서 `hhttps://...` → `https://...` 정정
- **참고**: 코드 변경 X, 사용자 환경 fix

### A-9. `.env` anon key 형식 잘못 — publishable key 사용

- **증상**: A-8과 같은 createClient 에러
- **원인**: 사용자가 Supabase Dashboard에서 새 형식 `sb_publishable_...` (46자) 복사. 우리 `supabase-js v2.46`이 기대하는 건 JWT 형식 anon key (`eyJ...` 200~250자)
- **fix**: 사용자가 Dashboard → API Keys → **legacy `anon` `public`** 행의 JWT를 `.env`에 입력
- **참고**: 코드 변경 X. supabase-js 최신 업그레이드 + publishable key 사용도 옵션이지만 v0.1.0 스코프 외

### A-10. `PrimaryButton` lg size — `h-13` 미존재 클래스

- **증상**: iPhone 화면에서 시작하기 버튼이 매우 얇게 렌더 (텍스트 + padding-y 0)
- **원인**: tailwind 기본 spacing scale + 우리 `tailwind.config.ts` `spacing` extend(`4/8/12/16/20/24/32/48/64`) 어디에도 `13` 없음 → `h-13` 클래스 무시됨 → 자연 height만 적용
- **fix**: `src/components/shared/primary-button.tsx`
  ```diff
    const HEIGHT: Record<Size, string> = {
  -   sm: 'h-9',
  -   md: 'h-11',
  -   lg: 'h-13',
  +   sm: 'h-[36px]',
  +   md: 'h-[44px]',
  +   lg: 'h-[52px]',
    };
  ```
- **root cause**: spec `<component_styling>.<primary_button>` (36/44/52px) ↔ 구현 매핑 시 tailwind scale 확인 누락

### A-11. `(tabs)/_layout.tsx` nested route가 별도 탭으로 자동 register

- **증상**: BottomNav에 5개 탭 외 추가로 **`cellar/index`**, **`cellar/[lwin]`**, **`settings/index`** 파일 경로 라벨로 노출됨 (스크린샷에서 직접 확인)
- **원인**: expo-router는 `(tabs)/cellar/_layout.tsx` 같은 nested layout 없으면 폴더 내 모든 route를 별도 Tab으로 자동 등록. `<Tabs.Screen name="cellar">` 명시했어도 children은 별도 처리됨
- **fix**: `app/(tabs)/_layout.tsx`에 explicit hide 추가
  ```tsx
  <Tabs.Screen name="cellar/index" options={{ href: null }} />
  <Tabs.Screen name="cellar/[lwin]" options={{ href: null }} />
  <Tabs.Screen name="settings/index" options={{ href: null }} />
  ```
- **더 깔끔한 root cause fix (TODO 다음 세션)**: `(tabs)/cellar/_layout.tsx`, `(tabs)/settings/_layout.tsx`에 Stack layout 두기 — explicit hide 대신 자연스러운 nested 구조

### A-12. `notes/new.tsx` BottomSheet dismiss 무한 루프

- **증상**: BottomSheet 위 dim 영역 탭하면 빠져나가지 못하고 계속 sheet 로딩됨
- **원인**: 수동 `<Pressable className="absolute inset-0" onPress={handleClose}>` 깔았는데 BottomSheet 컴포넌트와 zIndex 충돌. router.back() 호출되어도 sheet animation 없이 즉시 unmount → remount 루프
- **fix**: `@gorhom/bottom-sheet`의 표준 `BottomSheetBackdrop` 컴포넌트 사용
  ```diff
  - import BottomSheet, { BottomSheetView } from '@gorhom/bottom-sheet';
  + import BottomSheet, { BottomSheetView, BottomSheetBackdrop } from '@gorhom/bottom-sheet';

  + const renderBackdrop = useCallback(
  +   (props) => (
  +     <BottomSheetBackdrop
  +       {...props}
  +       appearsOnIndex={0}
  +       disappearsOnIndex={-1}
  +       pressBehavior="close"
  +       opacity={0.6}
  +     />
  +   ),
  +   [],
  + );

  - <View className="flex-1" style={{ backgroundColor: 'rgba(0,0,0,0.6)' }}>
  -   <Pressable className="absolute inset-0" onPress={handleClose} ... />
  + <View className="flex-1">
      <BottomSheet
        ...
  +     backdropComponent={renderBackdrop}
      >
  ```
- **root cause**: spec 단순화로 수동 Pressable 패턴 사용 — @gorhom/bottom-sheet 표준 패턴(`BottomSheetBackdrop`) 사용 권장으로 SKILL 보강 필요
