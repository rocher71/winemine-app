<overview>
winemine은 와인 라벨을 촬영하면 AI가 와인을 인식하고, 세계 지도에 방문 흔적을 시각화하며, 시음 노트를 beginner/expert 모드로 작성할 수 있는 모바일 앱이다. 사용자는 가입 없이 익명으로 시작해(Supabase Anonymous Auth) 즉시 사용할 수 있고, 나중에 Kakao/Google/Apple OAuth로 계정을 업그레이드해 기기 간 동기화·백업이 가능하다(v0.2.0 활성). 셀러(소장 와인) 관리·드링킹 윈도우 알림·5단계 레벨(L1 입문자 → L5 마스터)을 통해 와인 경험을 누적한다.

v0.1.0의 목적은 **스토어 심사 통과 가능한 최소 동작 앱**을 1주 안에 만드는 것이다. 12개 핵심 화면(온보딩 4 + 홈/캡처/와인 상세/셀러 리스트/셀러 상세/노트 출처 picker/노트 작성/노트 상세/설정 4)을 구현하고, 이미 Supabase에 존재하는 LWIN 기반 wines 카탈로그 + wine_korean_names 한글명 매핑을 그대로 활용하면서, 우리 앱이 추가로 필요한 메타데이터(bottle_color, drink_window)와 사용자 데이터(profiles, tasting_notes, cellar_items)를 신규 테이블로 추가한다. 라벨 인식은 Edge Function `label-scan`이 mock 응답(LWIN 고정)을 반환해 흐름 검증만 한다 — 실제 인식은 v0.2.0에서 **Gemini Vision API** 또는 **온디바이스 AI** (둘 중 비용·정확도 베타 비교 후 결정)로 통합.

CRITICAL: 비즈니스 로직의 80%는 Supabase SQL 함수 / RLS 정책에 배치해야 한다. Edge Functions는 **외부 API wrapper만** (label-scan, anonymize-id 등) 얇게 유지 — Free plan 단순성과 얇은 코드베이스를 유지하기 위함.

CRITICAL: 사용자 ID(Supabase UUID)는 절대 UI나 응답 body에 노출하지 않는다. 모든 사용자 식별자는 HMAC-SHA256 기반 익명화 패턴(`velvety-fox-37`)으로 변환 — 정책: [specs/domain/policies/anonymization.md](../../specs/domain/policies/anonymization.md).

CRITICAL: 모든 사용자 노출 텍스트는 LocalizedString `{ ko: string, en: string }` 양쪽을 채워야 한다. 영어 모드에서 한글 한 글자라도 노출되면 RC 차단. ko/en 누락은 lint로 검출 (key_implementation_notes 참조). 와인명은 예외 — `wine_korean_names`에 매핑이 없으면 ko 모드에서도 `wines.display_name` (영문) fallback 허용 (LWIN catalog 한계).

CRITICAL: 기존 `public.wines` 및 `public.wine_korean_names` 테이블은 외부 카탈로그 데이터다. v0.1.0에서 **schema 변경·데이터 INSERT/UPDATE/DELETE 금지** — 우리 앱 specific 정보는 `wine_metadata` 신규 테이블로 분리. `public.feedback`, `public.waitlist`는 기존 마케팅 사이트 자산이므로 v0.1.0 RN 앱에서 사용하지 않음.

CRITICAL: 모든 사용자 데이터 테이블(profiles, tasting_notes, cellar_items, wine_metadata-edits)에 RLS 활성화 필수. RLS 비활성 테이블 생성 금지.

CRITICAL: v0.1.0은 anonymous-only이지만 OAuth 업그레이드 구조를 미리 준비한다. profiles 테이블·auth provider 폴더·linkIdentity 호출 지점을 v0.1.0에서 미리 만들어, v0.2.0에서 Kakao/Google/Apple 추가가 0.5일 안에 끝나도록 한다.
</overview>

    - country_match: boolean (nullable)

    - llm_model: text (nullable)
    - llm_reasoning: text (nullable)
    - created_at: timestamptz

    RLS (이미 존재): `wine_korean_names_public_read` — SELECT to all

INSERT/UPDATE/DELETE는 service_role만 (v0.1.0은 트리거로 wines INSERT 시 row 자동 생성 X — wines 자체가 read-only이므로 wine_metadata는 빈 테이블로 시작하고, 필요한 LWIN에 대해서만 ingestion 파이프라인이 채움. v0.1.0에서는 type_canonical만이라도 일괄 seed: `INSERT INTO wine_metadata (lwin, type_canonical) SELECT lwin, 'red' FROM wines WHERE type = 'Wine'` 같은 단순 매핑)

    ```sql
    create view public.wines_localized as
    select
      w.lwin,
      w.display_name,
      kn.name_ko,
      w.producer_title,
      w.producer_name,
      w.wine,
      w.country,
      w.region,
      w.classification,
      w.type as type_raw,
      coalesce(wm.type_canonical, lower(split_part(w.type, ' ', 1))) as type_canonical,
      wm.bottle_color,
      wm.drink_window_from_year,
      wm.drink_window_peak_year,
      wm.drink_window_to_year,
      coalesce(wm.vintage_override,
        case when length(w.lwin) in (11, 13) then substring(w.lwin from 8 for 4)::int else null end
      ) as vintage,
      w.status
    from public.wines w
    left join lateral (
      select name_ko from public.wine_korean_names
      where lwin = w.lwin
      order by confidence desc nulls last, id asc
      limit 1
    ) kn on true
    left join public.wine_metadata wm on wm.lwin = w.lwin
    where w.status = 'Live';
    ```

    ```sql
    -- anonymize SQL 함수 (pgcrypto hmac)
    create or replace function public.anonymize(user_id uuid)
    returns text language plpgsql immutable as $$
    declare
      adjectives text[] := array['velvety','silky','crimson','ruby','amber','golden','smoky','dusky','radiant','sparkling']; -- 50개 권장 (specs/domain/policies/anonymization.md)
      nouns text[] := array['fox','bear','eagle','wolf','deer','owl','swan','hawk','panther','tiger'];
      h bytea;
      adj_idx int; noun_idx int; num int;
    begin
      h := extensions.hmac(user_id::text, current_setting('app.anonymization_salt', true), 'sha256');
      adj_idx := (get_byte(h, 0) % array_length(adjectives, 1)) + 1;
      noun_idx := (get_byte(h, 1) % array_length(nouns, 1)) + 1;
      num := ((get_byte(h, 2) * 256) + get_byte(h, 3)) % 100;
      return adjectives[adj_idx] || '-' || nouns[noun_idx] || '-' || num::text;
    end $$;
    -- salt는 ALTER DATABASE 또는 Supabase Vault로 설정

    create table public.profiles (
      id uuid primary key references auth.users(id) on delete cascade,
      anonymous_display text not null,
      email text,
      linked_providers text[] not null default '{}',
      is_upgraded boolean not null default false,
      mode text not null default 'first-time' check (mode in ('first-time','heavy')),
      experience text not null default 'beginner' check (experience in ('beginner','expert')),
      language text not null default 'ko' check (language in ('ko','en')),
      theme text not null default 'system' check (theme in ('system','dark','light')),
      xp int not null default 0 check (xp >= 0),
      level int not null default 1 check (level between 1 and 5),
      created_at timestamptz default now(),
      updated_at timestamptz default now()
    );
    alter table public.profiles enable row level security;
    create policy "profiles_select_own" on public.profiles for select using (id = auth.uid());
    create policy "profiles_update_own" on public.profiles for update using (id = auth.uid()) with check (id = auth.uid());

    -- 트리거: auth.users 생성 시 profiles 자동
    create or replace function public.handle_new_user() returns trigger language plpgsql security definer as $$
    begin
      insert into public.profiles (id, anonymous_display)
      values (new.id, public.anonymize(new.id));
      return new;
    end $$;
    create trigger on_auth_user_created
      after insert on auth.users for each row execute function public.handle_new_user();
    ```

    **20260519000400_wines_localized_view.sql**
    ```sql
    create view public.wines_localized
    with (security_invoker = true) as
    select
      w.lwin,
      w.display_name,
      kn.name_ko,
      w.producer_title, w.producer_name, w.wine,
      w.country, w.region, w.classification,
      w.type as type_raw,
      coalesce(wm.type_canonical, lower(split_part(w.type, ' ', 1))) as type_canonical,
      wm.bottle_color,
      wm.drink_window_from_year, wm.drink_window_peak_year, wm.drink_window_to_year,
      coalesce(wm.vintage_override,
        case when length(w.lwin) in (11,13) then substring(w.lwin from 8 for 4)::int else null end
      ) as vintage,
      w.status
    from public.wines w
    left join lateral (
      select name_ko from public.wine_korean_names
      where lwin = w.lwin
      order by confidence desc nulls last, id asc
      limit 1
    ) kn on true
    left join public.wine_metadata wm on wm.lwin = w.lwin
    where w.status = 'Live';

    grant select on public.wines_localized to anon, authenticated;

    -- wine_metadata 초기 seed: wines.type 기반 단순 매핑 (refine은 향후)
    insert into public.wine_metadata (lwin, type_canonical)
    select lwin,
      case
        when lower(type) like '%champagne%' or lower(type) like '%sparkling%' then 'sparkling'
        when lower(type) like '%white%' then 'white'
        when lower(type) like '%rose%' or lower(type) like '%rosé%' then 'rose'
        when lower(type) like '%fortified%' or lower(type) like '%port%' or lower(type) like '%sherry%' then 'fortified'
        when lower(type) like '%dessert%' or lower(type) like '%sweet%' then 'dessert'
        else 'red'
      end
    from public.wines
    on conflict (lwin) do nothing;
    ```
